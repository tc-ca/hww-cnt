Conduct an end-to-end demo

An end-to-end demo is a visual representation of the final product or service that will be delivered and how users will interact with it. Use this play to demonstrate products and services related to departmental operations.

Use this play to...

  • Visualize a solution, service, or product
  • Understand how the user will interact with it
  • Provide opportunities for feedback

Running the play

With so many pieces to any type of project, it’s easy to lose sight of the big picture. Often times, this can lead to integration issues on multiple levels – such as an IT solution not working with currently installed programs, or an HR innovation that doesn’t match up with the current method of hiring new employees.

Visualizing the deliverables of a project and how users will interact with it is key. This is done by using an end-to-end demo. The team can run through each step of the project, garnering feedback to make sure it works seamlessly with the existing structure upon delivery. Depending on the project, the demo may take the form of sketches or diagrams, a working prototype, or a new product. It is important to continually repeat the process and improve on it as it takes shape.

We recommend that this play be driven by the project owner, with core team members participating as necessary. A high-level decision maker can be involved in the first few sessions and any major milestone sessions afterwards.

Create the end-to-end demo

The end-to-end demo typically starts as a high-level, low-tech visualization of the project – such as a diagram or sketches. For a project that is an update to an existing product or service, the demo should also reflect how the work fits into the current experience.

As the project becomes more clearly defined, the demo will mature and grow. The end-to-end demo should be continually updated with new information and feedback. It’s critical to show how the user will discover the product, feature, or service, as well as how they interact with it and leave it.

Materials

To start this play, you will need: 

  • Paper 
  • Whiteboard 
  • Markers and pens 
  • Mock-ups of the project (User Interface, etc.) 

The demo

Set aside a 30-minute window with the project team as well as any stakeholders and high-level decision makers, if available. Walk through the entire demo, end-to-end, so everyone understands the user’s journey through the project and where it fits into the bigger picture. If this is not the first time going through the demo, you can place some focus on the parts that have changed – however it is most important to run through the demo in its entirety, and be sure to document all user feedback.

Tip

Your end-to-end demo should be rough. Don’t worry about creating a presentation or official document – the purpose is to quickly show how a product works in order to spark conversation.

Evolution

Make any necessary amendments to your demo based on user feedback. Consider presenting to a wide variety of test subject groups, such as other teams or individuals with the organization. By using a large test sample, especially one outside of the core project team, you may receive feedback that you might not have originally thought of.

If the feedback indicates that more significant changes are needed, then consider prototyping and re-running the demo to ensure the product satisfies it’s intended user. The most important takeaway, however, is once the project is implemented continue to follow up with users to identify and resolve any issues.

Tip

If your project is particularly big or complex, consider checking out the Map your users’ journey to gain the perspective of the end user.
